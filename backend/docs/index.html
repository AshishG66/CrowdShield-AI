CrowdShield API Reference

Welcome to the CrowdShield AI developer API docs. Integrate and manage telemetry readings, real-time alert dispatch systems, security logs, and venue metrics.

System APIs

GET /api/health
Retrieves the active health status, software build version, and timestamp of the backend service instance.
Query Parameters

None required.

Response (200 OK)
{
  "success": true,
  "message": "CrowdShield AI Backend Running",
  "version": "1.0.0",
  "timestamp": "2026-06-27T04:12:24.370Z"
}

Venues APIs

GET /api/venues
Retrieves the complete list of registered venues, containing geographic coordinates, gate statistics, and real-time active telemetry metrics.
Query Parameters

None required.

Response (200 OK)
{
  "success": true,
  "data": [
    {
      "_id": "60d5ec49f3183c26883e002a",
      "name": "M. Chinnaswamy Stadium",
      "type": "Stadium",
      "city": "Bengaluru",
      "capacity": 45000,
      "status": "Active",
      "sectors": []
    }
  ]
}
GET /api/venues/:id
Retrieves the full profile details of a single venue using its unique database record identifier.
Path Parameters
Param Type Required Description
id string Yes The Mongo ObjectId identifier.
Response (200 OK)
{
  "success": true,
  "data": {
    "_id": "60d5ec49f3183c26883e002a",
    "name": "M. Chinnaswamy Stadium",
    "type": "Stadium",
    "capacity": 45000,
    "latitude": 12.9784,
    "longitude": 77.5994,
    "entryGates": 8,
    "exitGates": 8,
    "emergencyExits": 4,
    "status": "Active"
  }
}
POST /api/venues
Registers a new venue profile in the CrowdShield database. Input payload is validated strictly by Zod request middleware schemas.
Request Body Fields
Field Type Req Desc
name string Yes Unique name.
type string No Default: "Stadium".
city string Yes Location city.
state string Yes Location state.
capacity number Yes Max capacity limit.
latitude number Yes Map coordinate.
longitude number Yes Map coordinate.
Response (201 Created)
{
  "success": true,
  "message": "Venue created successfully",
  "data": {
    "_id": "60d5ec49f3183c26883e002a",
    "name": "M. Chinnaswamy Stadium",
    "capacity": 45000
  }
}
PUT /api/venues/:id
Modifies an existing venue profile using field-level parameters passed in request bodies. Object keys not supplied remain unchanged.
Path Parameters
Param Type Req Desc
id string Yes The Mongo ObjectId.
Response (200 OK)
{
  "success": true,
  "message": "Venue updated successfully",
  "data": {
    "_id": "60d5ec49f3183c26883e002a",
    "name": "M. Chinnaswamy Stadium",
    "status": "Inactive"
  }
}
DELETE /api/venues/:id
Permanently removes a venue profile registry from the database using its unique identifier.
Path Parameters
Param Type Req Desc
id string Yes The Mongo ObjectId.
Response (200 OK)
{
  "success": true,
  "message": "Venue deleted successfully"
}
